# **Xeni 项目背景与研发蓝图**

## **一、 Xeni 1.0：现状 - 核心自动化链路**

目前 Xeni 已实现从文档到表单的基础自动化闭环，核心在于“数据搬运”的效率提升。

### **1.1 核心流程**

- **案件创建：** 律师创建 Case 并选择对应的签证种类（Visa Type）。
- **信息采集：** 录入申请人基本信息，上传护照及各类证明材料（Supporting Documents）。
- **AI 提取：** 系统开启文档分析，根据预设的 **Schema 结构**提取关键字段信息。
- **自动填表：** **Form Pilot** 调用提取的结构化信息，自动在政府官网完成在线表格填写。

### **1.2 核心价值**

- **提效：** 将律师从 PDF 与网页间的“复制粘贴”中解放出来。
- **标准化：** 建立结构化的案件知识库，取代传统的文件夹管理模式。

------

## **二、 Xeni 2.0：未来 - 全流程风控与协作**

2.0 阶段的重点是在自动化链路之前增加“质量与合规”的过滤网，并引入外部协作终端。

### **2.1 新增模块一：Intake Manager（前置质量风控）**

在 AI 提取信息前，确保“原材料”的完整性与合规性。

- **智能 Checklist：** 根据 Visa Type 预设清单，自动检查文档是否缺失、是否符合上传规范。
- **质量 Issue 汇总：** 实时反馈模糊、过期或类型错误的文档，让律师/申请人有的放矢地解决问题。
- **文档加工工具：** * **合并：** 支持将多份 PDF（如流水）整合为单一文件。
  - **压缩：** 自动处理文件体积，确保符合政府官网上传限制。

### **2.2 新增模块二：Compliance Engine（深度逻辑审计）**

在模块一检查通过后启动，聚焦于“逻辑与语义”层面的冲突识别。

- **语义冲突识别：** 交叉比对不同文档间的关键要素（如姓名拼写、日期冲突等）。
- **业务规则检查：** 验证如银行账单金额是否满足签证特定要求等深度逻辑。
- **逻辑 Issue 汇总：** 在填表前消除因信息不自洽导致的拒签隐患。

### **2.3 新增终端：Applicant Portal（申请人门户）**

将申请人引入协作流，降低律师的沟通成本。

- **进度实时可见：** 申请人可自主查看 Case 的实时办理状态。
- **自助补件协作：** 申请人可直接查看律师要求的文档清单、上传文件并根据 Issue 反馈修复文档。

## **三、 核心概念对比表**

| **特性**     | **1.0 现状 (Efficiency)** | **2.0 规划 (Quality & Collaboration)**                  |
| ------------ | ------------------------- | ------------------------------------------------------- |
| **关注点**   | 自动提取、快速填表        | 补件协作、文档质量、逻辑合规                            |
| **核心组件** | AI Parser, Form Pilot     | **Intake Manager, Compliance Engine, Applicant Portal** |
| **文档处理** | 原始文档直接提取          | **格式化（合并/压缩）、预设 Checklist 检查**            |
| **反馈机制** | 无或人工发现错误          | **Issue 实时汇总系统（质量类 + 逻辑类）**               |
| **用户角色** | 仅律师端                  | **律师端 + 申请人端 (Portal)**                          |
